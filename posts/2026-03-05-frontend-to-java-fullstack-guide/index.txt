<h2>为什么前端要学 Java？</h2>
<p>作为一名前端工程师，你可能有过这些想法：</p>
<ul>
<li>&quot;只会前端，职业发展受限，想转型全栈&quot;</li>
<li>&quot;后端工资更高，但不知道从哪入手&quot;</li>
<li>&quot;听说过 Spring Boot，但不知道和 Express 有什么区别&quot;</li>
<li>&quot;想自己独立做项目，不想依赖后端同事&quot;</li>
</ul>
<p>2026 年的今天，Java 依然是企业级应用的首选语言。掌握 Java + Spring Boot，意味着：</p>
<ul>
<li>✅ 能够独立开发完整的 Web 应用</li>
<li>✅ 职业选择更广，可应聘全栈/后端岗位</li>
<li>✅ 薪资上限更高（全栈工程师平均薪资比纯前端高 30%+）</li>
<li>✅ 更好地理解系统架构，反哺前端技术</li>
</ul>
<p><strong>关键问题：前端转 Java 难吗？</strong></p>
<p>说实话，有前端经验的人学 Java 有天然优势：</p>
<ol>
<li><strong>编程思维已建立</strong>：变量、函数、循环这些概念都是相通的</li>
<li><strong>TypeScript 是桥梁</strong>：如果你用过 TS，Java 的类型系统会让你感到熟悉</li>
<li><strong>HTTP 协议已掌握</strong>：REST API、状态码、请求方法你已经很熟了</li>
<li><strong>AI 工具加持</strong>：现在有大量 AI 工具可以辅助学习和调试</li>
</ol>
<p>我设计这套学习路径的目标是：<strong>让有前端经验的开发者，用 12 周时间系统掌握 Java 全栈开发</strong>。</p>

<h3>Day 6-10：面向对象编程</h3>
<p>这是 Java 的核心，也是前端开发者需要重点理解的部分。</p>
<p><strong>类的定义</strong>：</p>
<pre><code class="language-java">public class User {
    // 属性（类似 TS 的类属性）
    private Long id;
    private String name;
    private String email;

    // 构造方法（类似 TS 的 constructor）
    public User(String name, String email) {
        this.name = name;
        this.email = email;
    }

    // 方法
    public void sayHello() {
        System.out.println(&quot;Hello, I&#39;m &quot; + name);
    }

    // Getter/Setter（Java 特有，用于封装）
    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }
}
</code></pre>
<p><strong>继承与多态</strong>：</p>
<pre><code class="language-java">// 父类
public class Animal {
    public void makeSound() {
        System.out.println(&quot;动物发声&quot;);
    }
}

// 子类
public class Dog extends Animal {
    @Override
    public void makeSound() {
        System.out.println(&quot;汪汪汪！&quot;);
    }
}

// 使用
Animal myDog = new Dog();  // 父类引用，子类对象
myDog.makeSound();  // 输出：汪汪汪！
</code></pre>
<p><strong>接口</strong>（类似 TypeScript Interface）：</p>
<pre><code class="language-java">public interface Flyable {
    void fly();
}

public class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println(&quot;鸟儿在飞&quot;);
    }
}
</code></pre>

class Library {

<h3>第一个 REST Controller</h3>
<pre><code class="language-java">@RestController
@RequestMapping(&quot;/api/hello&quot;)
public class HelloController {

    // GET /api/hello
    @GetMapping
    public String sayHello() {
        return &quot;Hello, Spring Boot!&quot;;
    }

    // GET /api/hello/{name}
    @GetMapping(&quot;/{name}&quot;)
    public String sayHelloToName(@PathVariable String name) {
        return &quot;Hello, &quot; + name + &quot;!&quot;;
    }

    // GET /api/hello/greeting?name=John
    @GetMapping(&quot;/greeting&quot;)
    public String greeting(@RequestParam String name) {
        return &quot;Good morning, &quot; + name + &quot;!&quot;;
    }

    // POST /api/hello
    @PostMapping
    public ResponseEntity&lt;String&gt; create(@RequestBody Map&lt;String, String&gt; body) {
        String name = body.get(&quot;name&quot;);
        return ResponseEntity.ok(&quot;Created: &quot; + name);
    }
}
</code></pre>
<p>看到 <code>@GetMapping</code>、<code>@PostMapping</code> 这些注解，是不是想到了 Express 的路由？</p>
<h3>依赖注入（DI）：Spring 的核心</h3>
<pre><code class="language-java">// Service 层
@Service
public class UserService {

    @Autowired  // 自动注入
    private UserRepository userRepository;

    public List&lt;User&gt; getAllUsers() {
        return userRepository.findAll();
    }
}

// Controller 层
@RestController
@RequestMapping(&quot;/api/users&quot;)
public class UserController {

    @Autowired
    private UserService userService;

    @GetMapping
    public List&lt;User&gt; getAllUsers() {
        return userService.getAllUsers();
    }

<h3>JWT 认证流程</h3>
<pre><code>1. 用户登录 → 发送账号密码
2. 服务器验证 → 生成 JWT Token
3. 客户端存储 → localStorage
4. 后续请求 → Header 携带 Token: Bearer xxx
5. 服务器验证 → 解析 Token 获取用户信息
</code></pre>
<h3>JWT 工具类</h3>
<pre><code class="language-java">@Component
public class JwtUtil {

    @Value(&quot;${jwt.secret}&quot;)
    private String secret;

    public String generateToken(UserDetails userDetails) {
        return Jwts.builder()
            .setSubject(userDetails.getUsername())
            .setIssuedAt(new Date())
            .setExpiration(new Date(System.currentTimeMillis() + 86400000))
            .signWith(SignatureAlgorithm.HS256, secret)
            .compact();
    }

    public Claims parseToken(String token) {
        return Jwts.parser()
            .setSigningKey(secret)
            .parseClaimsJws(token)
            .getBody();
    }
}
</code></pre>
<h3>认证 Controller</h3>
<pre><code class="language-java">@RestController
@RequestMapping(&quot;/api/auth&quot;)
public class AuthController {

    @Autowired
    private AuthenticationManager authenticationManager;

    @Autowired
    private JwtUtil jwtUtil;

    @PostMapping(&quot;/login&quot;)
    public ResponseEntity&lt;?&gt; login(@RequestBody LoginRequest request) {
        authenticationManager.authenticate(
            new UsernamePasswordAuthenticationToken(
                request.getUsername(),
                request.getPassword()
            )
        );

        String token = jwtUtil.generateToken(request.getUsername());
        return ResponseEntity.ok(new LoginResponse(token));
    }
}
